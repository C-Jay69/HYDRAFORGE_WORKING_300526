# M&A Merger Risk Analyzer — Python Engine

Standalone Python scoring engine for the Hydraforge M&A review platform.

## Files

| File | Description |
|------|-------------|
| `merger_risk_analyzer.py` | Core `MergerRiskAnalyzer` class — consumes YAML config |
| `test_merger_analyzer.py` | Unit test suite — 5 sample agreements with expected scores |
| `api.py` | FastAPI production server |
| `openapi.yaml` | OpenAPI 3.0 schema |
| `requirements.txt` | Python dependencies |
| `DEPLOY.sh` | Deployment commands |

## Setup

```bash
pip install -r requirements.txt
```

## Run tests

```bash
python test_merger_analyzer.py
```

> Note: tests require `merger_scoring_config.yaml` in the working directory.
> Copy it from the YAML config generated by prompt-3.

## Run API server

```bash
python api.py
# → http://localhost:8000
# → http://localhost:8000/docs  (Swagger UI)
```

## Relationship to TypeScript app

This Python engine is the **standalone/CLI** version of the scoring logic.
The TypeScript app (`packages/web`) embeds the scoring rules directly in
`openrouter.ts` via the LLM prompt + `validateScore()` server-side validator.

Both implement the same scoring rubric from `merger_scoring_config.yaml`.
